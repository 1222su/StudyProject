
## 常用git操作对应vscode
 Git 命令                | 中文版 VS Code 里的「按钮 / 入口」   | 快捷键 / 位置提示                             |
| --------------------- | ------------------------- | -------------------------------------- |
| `git init`            | 源代码管理 → 「初始化存储库」          | 左侧活动栏第三个图标（分叉图标）                       |
| `git clone`           | 命令面板 → 「Git: 克隆」          | `F1` 或 `Ctrl+Shift+P` → 输入 `git clone` |
| `git status`          | 打开「源代码管理」面板即可             | 侧边栏直接看文件列表和字母（U、M、A…）                  |
| `git add 文件`          | 文件右侧的 `+` 号（「暂存更改」）       | 鼠标悬停显示「暂存更改」                           |
| `git add .`           | 「更改」区块右侧「**全部暂存**」按钮      | 或 `Ctrl+Enter` 后选择“全部暂存”               |
| `git commit -m "xxx"` | 上方输入框填消息 → 按 `Ctrl+Enter` | 输入框提示「消息 (按 Ctrl+Enter 提交)」            |
| `git push`            | 底部状态栏「**发布分支**」或「**同步**」↻ | 首次推送显示「发布分支」，后续点「同步」                   |
| `git pull`            | 同上「同步」↻                   | 拉和推合并在一个按钮                             |
| `git checkout -- 文件`  | 文件右侧 ↩️ 「放弃更改」            | 鼠标悬停显示「放弃更改」                           |
| `git reset HEAD 文件`   | 已暂存文件右侧 `-` 「取消暂存」        | 在「STAGED CHANGES」区域                    |
| `git branch`          | 左下角分支名点击 → 可查看所有分支        | 状态栏 `main*` 字样                         |
| `git checkout 分支`     | 左下角分支名 → 选择要切换的分支         | 命令面板也能搜「Git: 签出到…”                     |
| `git log`             | 右侧「时间轴」或装 Git Graph 插件    | 右键文件 → Git Graph / Git History         |



## 字母对应格式

| 字母    | 全称        | 中文       | 出现位置              |
| ----- | --------- | -------- | ----------------- |
| **U** | Untracked | 未跟踪（新文件） | 资源管理器绿色文件名、SCM 列表 |
| **M** | Modified  | 已跟踪但被修改  | 橙色文件名             |
| **A** | Added     | 已暂存的新文件  | STAGED CHANGES 列表 |
| **D** | Deleted   | 删除       | 文件名带红叉            |
| **R** | Renamed   | 重命名      | 旧文件 D，新文件 R       |
